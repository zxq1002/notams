  <action>
    Refactor all fetcher modules to be more resilient and configurable.
    - Add `FETCH_TIMEOUT = 20` to `config.py`.
    - Update all `requests` calls in the fetcher files to use `timeout=config.FETCH_TIMEOUT` (or a local override).
    - Ensure `response.raise_for_status()` is called after requests.
    - Wrap the main logic of each fetcher in a `try-except` block.
    - Fetchers should return a standardized empty result (matching the expected schema) or include an "error" key when an exception occurs, rather than crashing.
  </action>

  </verify>

<task type="auto">
  <name>Task 2: Graceful Error Propagation in Server Routes</name>
  <files>service/server.py</files>
  <action>
    Update `/fetch` and `/fetch_archive` routes to handle fetcher errors gracefully.
    - In `/fetch`, wrap individual fetcher calls in `try-except` to ensure one failure doesn't block others.
    - Aggregate error messages from failed fetchers.
    - If ALL attempted fetchers fail, return a 500 error with a JSON body describing the failure.
    - In `/fetch_archive`, ensure exceptions are caught and returned as a 500 JSON error.
    - Standardize the error response format: `{"error": "Description of failure"}`.
  </action>
  <verify>
    Temporarily break a fetcher (e.g., by changing its URL to an invalid one) and verify that the `/fetch` route still returns results from other working fetchers and logs the error.
  </verify>
  <done>Flask routes robustly handle fetch failures and propagate errors to the UI.</done>
</task>
